React API

memo

memo 允许你的组件在 props 没有改变的情况下跳过重新渲染。

PureComponent

PureComponent 类似于 Component，但是当 props 和 state 与之前保持一致时会跳过重新渲染。

PureComponent 是 Component 的子类，并且支持 所有 Component 的 API。继承 PureComponent 的子类相当与定义了一个自定义的 shouldComponentUpdate 方法，该方法将浅比较 props 和 state。

isValidElement

用于检测参数值是否为 React 元素。

React 元素指的是：

  • 通过 JSX 标签 产生的值。
  • 通过调用 createElement 产生的值。

其他任意值，例如字符串，数字或者任意对象和数组等都不是 React 元素。

React 元素 VS React 节点

React 节点可能是：

  • 像 <div /> 或者 createElement(‘div’) 这样被创建的 React 元素
  • 使用 createPortal 创建的 portal
  • 字符串
  • 数字
  • true、false、null 或者 undefined（不会被展示出来）
  • 其他 React 节点的数组

所以 isValidElement 只能检测参数是否为 React 元素，而不能检测它是否是 React 节点。例如 42 不是一个有效的 React 元素，但它是一个有效的 React 节点。
