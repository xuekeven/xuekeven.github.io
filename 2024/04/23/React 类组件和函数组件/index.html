React 类组件和函数组件

ref

使用 ref 的情形：

  • 当希望组件“记住”某些信息，但又不想让这些信息 触发新的渲染
  • 操作 React 组件的 DOM

创建

  • 类组件使用 createRef
  • 函数组件使用 useRef

消费

类组件和函数组件消费 ref 不同主要体现在操作子组件的 DOM：

  • 可以直接读取类组件所有内容。
  • 需要 forwardRef 包裹才能读取函数组件 useImperativeHandle 里的内容。

Context

通常来说，你会通过 props 将信息从父组件传递到子组件。但是，如果你必须通过许多中间组件向下传递 props，或是在你应用中的许多组件需要相同的信息，传递 props 会变的十分冗长和不便。Context 允许父组件向其下层无论多深的任何组件提供信息，而无需通过 props 显式传递。

创建

类组件和函数组件：

  • 都用 createContext 创建 Context。
  • 都用 SomeContext.Provider 包裹子组件和提供 Context 的值。

消费

  • 类组件只能使用 SomeContext.Consumer 消费。
  • 函数组件可使用 SomeContext.Consumer 或者 useContext 消费。
