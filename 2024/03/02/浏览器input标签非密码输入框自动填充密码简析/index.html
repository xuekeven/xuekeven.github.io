浏览器input标签非密码输入框自动填充密码简析

背景

如下图，点击第二个下拉选择框后，浏览器自动提示出现密码管理器中的保存的密码进行填充。而且，可以看到下拉框用到的 input 并没有设置 type 为 password ，而是默认的 text ，我们不禁发出一声：“ Chrome 老师，发生甚么事了？”

分析

经过上网查询和分析，发现问题出在下面的一个 type 为 password 的 input。

是不是还是满脸问号？其实，问题还就是出在 Chrome 浏览器上。

因为开启了浏览器的自动填充密码后，浏览器找密码区域很容易，只需要知道哪个 input 的 type 是 password 即可，那么它怎么知道谁是用户名呢？

答案是，浏览器默认会去找离 type=”password” 的 input 最近的上方的那个 input 为“用户名”进行填充，只要一获取上面 input 的焦点，就会自动出现 “用户名填充区域”。

这种情况发生在 点击了type=”password” 的 input后，再去点击离 type=”password” 的 input 最近的上方的那个 input的时候。

如果是，上来没有先点 type=”password” 的 input ，而是先选了下拉框，此时就不会出现这种问题。不过，等你填了 type=”password” 的 input ，如果再去上面选，还会出现这个问题。

解决

如果，下面我只需要一个 type=”password” 的 input 输入框，不需要有 “用户名” ，并且不想影响上面的下拉框，应该怎么办呢？

  • 1、禁用掉浏览器的自动填充功能。

直接从根源解决问题，不过管不了用户的浏览器设置，pass。

  • 2、给 type=”password” 的input设置属性 autocomplete=”off”。

查询 MDN 可知，浏览器根本不鸟”off“值，pass。详见：autocomplete 属性和登录字段

  • 3、给 type=”password” 的 input 设置属性 autocomplete=”new-password” 。

查询 MDN 可知，浏览器基本也都不鸟 “new-password” 值，pass。详见：使用 autocomplete=”new-password” 阻止自动填充

  • 4、自己在 type=”password” 的 input 的上面加一个不用的”用户名“ input 。

没办法的办法，目前看只能这样了，如下面类名为”hidden-input“的input标签：
但是要注意，如果你给这个不用的”用户名“ input 设置 display:”none” ，你会发现，浏览器真聪明！点击上面的下拉框还是会出现自动填充^_^。
所以，不能设置 display:”none” ，可以用固定定位或相对定位后，设置 left（如设置为99999px），再加上 opacity 为 0。
不过，这个方法还有一个缺点，就是，浏览器还是会自动填充 type=”password” 的 input ，不过也管不了那么多了。

  • 5、将 type=”password” 的 input 改为type=”text“，并使用 passwordEntry.ttf。

将 input 的 type 改为 text 后，给 input 设置 css 的 font-family 为 passwordEntry.ttf ，而 passwordEntry 不是常见的字体，而是把所有的字体全部显示成原点。这样就模拟了密码输入。ttf 的兼容性也十分好，不过我没有试和用这种方法。
